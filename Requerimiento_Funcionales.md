# Requerimientos Funcionales

## RF-01 – Registro de usuarios  
El sistema debe permitir la creación de cuentas mediante un formulario con nombre, correo y contraseña.

## RF-02 – Autenticación  
El sistema debe validar credenciales y permitir el inicio de sesión.

## RF-03 – Registro de solicitudes  
El usuario debe poder registrar tickets indicando categoría, descripción, prioridad y evidencias opcionales.

## RF-04 – Asignación de solicitudes  
El coordinador debe poder asignar solicitudes a técnicos disponibles.

## RF-05 – Actualización de estado  
El técnico debe poder actualizar el estado de un ticket: Pendiente, En proceso, Resuelto, Cerrado.

## RF-06 – Consulta de estado  
Los usuarios deben poder consultar el estado de sus solicitudes en cualquier momento.

## RF-07 – Gestión de prioridades  
El sistema debe permitir clasificar solicitudes según prioridad (Alta, Media, Baja).

## RF-08 – Historial de acciones  
El sistema debe registrar cada acción realizada sobre una solicitud para garantizar trazabilidad.

## RF-09 – Notificaciones  
El sistema debe enviar notificaciones cuando una solicitud cambie de estado o prioridad.

## RF-10 – Panel para técnicos  
Cada técnico debe poder visualizar únicamente los tickets asignados a él.

## RF-11 – Reportes  
El sistema debe generar reportes por estado, técnico, tiempos de respuesta y categorías.

## RF-12 – Gestión de roles y usuarios  
El coordinador debe administrar roles como estudiante, docente, administrativo, técnico y coordinador.
